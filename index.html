Signal Board

Signal Board is a Node.js/Express + PostgreSQL feature voting API — a deliberately engineered backend built as a performance engineering lab. Users submit feature requests and vote on priorities; the architecture demonstrates the full arc from N+1 anti-pattern diagnosis to a quantified 22x speedup, documented via ADRs at every step.

4-layer architecture · Zod validation · Redis: 282ms → 2ms · ADR docs

Key Features

Layered Architecture
Strict four-layer separation — routes → controllers → services → db — with Zod validation at the HTTP boundary. Each layer has one responsibility; services are testable without a database or HTTP server.
N+1 Diagnosis → 22x Speedup
Diagnosed an intentional N+1 anti-pattern (501 queries, ~5856ms) using EXPLAIN ANALYZE and structured logs. Replaced with a single LEFT JOIN — 1 query, ~263ms. LEFT JOIN over INNER JOIN was a deliberate correctness call: INNER JOIN silently drops zero-vote requests.
Centralized Error Handling
Custom AppError class + global middleware ensure consistent { error, code, details } response shapes. Intentional errors surface to clients; unexpected errors log server-side behind the correlation ID, preventing information leakage.

Language Hackalator

Language Hackalator is a Chrome extension built for language learners reading foreign-language content online. Designed, architected, and shipped independently from manifest configuration to accessible tooltip UI.

Demo of low-friction contextual translation

Key Features

Floating Tooltip Translation
A content script detects text selection on any webpage and fetches live translations from the MyMemory API, rendering results in a scoped, auto-dismissing tooltip anchored near the highlighted text.
MV3 Chrome Extension Architecture
Built on Manifest V3 with a popup-based language selector, chrome.storage.local persistence, and injected styles scoped to prevent host-page conflicts — no server, no API key required.
WCAG 2.1 AA Accessibility
Tooltip announces via aria-live="polite" for screen readers; dismisses via Escape key, outside click, or timeout; keyboard focus management implemented throughout — all absent from naive tooltip patterns.

"Cache Me If You Can"

Systems Design · Redis Architecture · Cognitive Load as a Lens

✓ Delivered — public meetup · view tech talk slides →

A 12-minute tech talk sponsored by Pressable exploring Redis architecture through the lens of cognitive systems design — how human memory constraints map directly onto how Redis manages latency, TTL, eviction, and persistence. Delivered to engineers at mixed seniority levels.

Topics Covered

CS Meets Neuroscience
Redis and neuroscience are solving the same problem: how do you move the right data to the right place, given that memory is finite? The Atkinson-Shiffrin model encodes information through short-term into long-term storage — Redis is engineered around the same constraint.
TTL + the Forgetting Curve
Time-to-Live as a model for both application session management and how humans naturally shed short-term memory under load. Contraindications covered: audit requirements, debugging tools.
Cache Stampede Failure Mode
The 2010 Facebook incident: cache-miss overload that caused a 2.5-hour outage. Used as a case study in how architectural assumptions — not Redis itself — cause failures at scale.

REDIS + Neuroscience teach us this: limits aren’t flaws-they’re design opportunities. So . . . design for differences – in systems and people.

Component-Based Bilingual Learning Prototype

A Figma component library and prototype demonstrating the design thinking behind a new McGraw Hill product build.

Engineering Decisions

Variant-Based Component Architecture
Single-source system with DOK 1–3 variants and bilingual configurations — consistency across 50+ lessons without divergent implementations.
Shift-Left Accessibility
WCAG 2.1 AA built into every variant at design time. Contrast, semantic structure, and interactive states verified before a line of code is written.
Cognitive Load as a Constraint
Side-by-side bilingual scaffolding and progressive DOK complexity treat cognitive load as an architectural constraint, not a UX afterthought.
