Language Hackalator

Language Hackalator is a Chrome extension built for language learners reading foreign-language content online. Designed, architected, and shipped independently from manifest configuration to accessible tooltip UI.

Demo of fully accessible mouse and keyboard nav

Key Features

Floating Tooltip Translation A content script detects text selection on any webpage and fetches live translations from the MyMemory API, rendering results in a scoped, auto-dismissing tooltip anchored near the highlighted text.
MV3 Chrome Extension Architecture Built on Manifest V3 with a popup-based language selector, chrome.storage.local persistence, and injected styles scoped to prevent host-page conflicts — no server, no API key required.
WCAG 2.1 AA Accessibility Tooltip announces via aria-live="polite" for screen readers; dismisses via Escape key, outside click, or timeout; keyboard focus management implemented throughout — all absent from naive tooltip patterns.

CERES Discovery Sprint

I don't just use the tools my engineering team builds — I investigate them. A structured product discovery sprint on McGraw Hill's internal content authoring platform, applying the six-part Pendo Product Discovery Framework to surface usability and feasibility risks and translate them into an engineering recommendation.

Active · Interviews begin [INSERT DATE] · Full documentation linked below

Key Features

Generative User Research Semi-structured interviews with 4–6 instructional designers across content areas, designed to surface the context, pain, and workarounds that quantitative data misses.
Pendo Product Discovery Framework All six framework stages mapped explicitly to sprint phases — from defining the business outcome through affinity mapping, HMW statements, and closing the loop with an engineering deliverable.
Bias Awareness Protocols Confirmation bias, authority bias, and bandwagon effect each addressed by design: open-ended questions, equal participant weighting, and frequency + severity scored independently in synthesis.
Engineering-Facing Output The deliverable is not a manager report. It's a prioritized recommendation document written for the Content Authoring Engineering team — actionable, risk-tagged, and Impact vs. Effort ranked.
Jobs to Be Done Framework Every interview question is built around one guiding question: what job are authors hiring this tool to do — and where does the tool get fired and a workaround gets hired instead.

The deliverable isn't a list of complaints. It's a prioritized recommendation written for engineers.
